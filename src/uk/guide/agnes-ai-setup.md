# Налаштування Agnes AI

Цей посібник допоможе налаштувати вбудований AI Agent у XMCL.

::: tip Agnes AI безкоштовний
**Agnes AI повністю безкоштовний.** Достатньо створити безкоштовний API key —
жодної оплати, підписки чи банківської картки не потрібно.
:::

Після налаштування ви зможете:

- відкривати вікно AI Agent;
- аналізувати crash-логи вбудованим агентом;
- використовувати інструменти агента (увімкнення/вимкнення модів, встановлення, пошук/редагування конфігів тощо).

## Що потрібно

- Актуальна версія XMCL.
- Доступ до інтернету.
- Безкоштовний Agnes AI API key.

## Крок 1: Отримайте Agnes AI API key

1. Відкрийте портал розробника Agnes AI.
2. Увійдіть або створіть обліковий запис.
3. Створіть API key.
4. Скопіюйте та збережіть ключ у безпечному місці.

![Сторінка консолі Agnes AI, де створюється/копіюється API key](create-and-copy.webp)

## Крок 2: Відкрийте налаштування Agent у XMCL

1. Відкрийте XMCL.
2. Перейдіть у **Settings -> General**.
3. Увімкніть **Developer Mode**.
4. Прокрутіть до секції **AI Agent**.

![Загальні налаштування XMCL з виділеними Developer Mode та секцією AI Agent](general-setting.webp)

## Крок 3: Заповніть поля Agent

У секції **AI Agent**:

- **API Key**: вставте Agnes key;
- **Model**: залиште значення за замовчуванням, якщо немає інших вимог;
- **Endpoint**: залиште значення за замовчуванням, якщо немає інших вимог.

Стандартний Agnes endpoint:

```text
https://apihub.agnes-ai.com/v1/chat/completions
```

![Форма AI Agent з полями API Key / Model / Endpoint і прикладами значень](general-setting.webp)

## Крок 4: Перевірте, що все працює

1. Натисніть `Ctrl/Cmd + Shift + A`, щоб відкрити Agent.
2. Надішліть простий запит, наприклад: `list my installed mods`.
3. Переконайтеся, що агент відповідає без помилки конфігурації.

## Усунення проблем

### Agent не відкривається

- Перевірте, що увімкнено **Developer Mode**.
- Після увімкнення Developer Mode перезапустіть XMCL.

### Все ще показується попередження "не налаштовано"

- Перевірте API key (без зайвих пробілів/переносів рядка).
- Переконайтеся, що endpoint доступний із вашої мережі.

### Помилка запиту / 401 / 403

- API key недійсний, прострочений або без потрібних прав.
- Згенеруйте новий ключ у порталі Agnes і вставте його знову.

### Таймаут запиту

- Перевірте фаєрвол/проксі.
- Спробуйте ще раз зі стандартним endpoint.

## Безпека

- Ставтеся до API key як до пароля.
- Не публікуйте скріншоти з видимим ключем.
- Якщо підозрюєте витік, одразу замініть ключ.
