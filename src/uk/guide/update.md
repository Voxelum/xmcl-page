# Посібник з оновлення

> В розробці

Оскільки лаунчер все ще перебуває на стадії `beta`, резервне копіювання даних є особливо важливим... Тут ми розповімо, як вирішити проблеми під час оновлення та як зробити резервну копію перед перевстановленням.

## Оновлення APPX вимагає перевстановлення

Оскільки стара версія лаунчера у форматі `APPX` має застарілі сертифікати, під час оновлення зі старої версії на нову потрібно буде видалити стару версію. У цьому випадку, будь ласка, зробіть резервну копію папки даних XMCL.

Припустимо, що `foo` — це ваше ім'я користувача:

::: code-group
```[Windows (APPX)]
%LocalAppData%\Packages\XMCL_ncdvebj03zfcm\LocalCache\Roaming\xmcl
```
:::

Ви можете скопіювати цю папку і зберегти її, наприклад, на робочому столі. Після перевстановлення нова версія XMCL використовуватиме такий шлях:

::: code-group
```[Windows (APPX)]
%LocalAppData%\Packages\XMCL_68mcaawk44tpj\LocalCache\Roaming\xmcl
```
:::

Просто поверніть збережені файли резервної копії назад у цю папку.

## Помилка оновлення APPX

При оновленні APPX має відкритися папка, де знаходиться завантажений файл `xmcl.appinstaller`. Ви можете вручну двічі клацнути цей файл, щоб встановити оновлення.

## Інші оновлення вимагають перевстановлення

У цьому випадку насправді робити резервну копію необов'язково, оскільки тільки `APPX` очищує файли даних XMCL. Звичайно, ви можете підстрахуватися та скопіювати їх. Файли даних розташовані в таких місцях:

::: code-group
```cmd [Windows]
%AppData%\xmcl
```
```sh [macOS]
~/Library/Application Support/xmcl
```
```sh [Linux]
~/.config/xmcl
```
:::
