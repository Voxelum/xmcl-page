# Орнату және іске қосу ақаулықтарын жою

Егер сізде Minecraft, мод жүктегіштерді (Forge/Fabric/NeoForge/Quilt), модтарды, модпактарды, шейдерлерді орнату кезінде немесе ойынды іске қосу кезінде мәселелер туындаса, бұл нұсқаулық сізге ақауларды кезең-кезеңімен табуға және жоюға көмектеседі.

---

## 🌐 1. Жүктеудің сәтсіз аяқталуы немесе тұрып қалуы (Желі мәселелері)

### Симптомдары
* Minecraft файлдарын, ресурстарды, кітапханаларды немесе Forge/Fabric жүктеу `0%` деңгейінде тұрып қалады.
* Лаунчер байланыс үзілуі немесе уақыт біту қателерін көрсетеді (`CONNECTION_TIMED_OUT`, `NAME_NOT_RESOLVED`, `HTTP_STATUS 504`).

### Шешімі

:::tip Балама жүктеу көзін (Mirror) пайдаланыңыз
Егер ресми Mojang немесе Forge серверлері баяу жұмыс істесе немесе сіздің интернет провайдеріңіз тарапынан блокталған болса, балама жүктеу көзіне ауыса аласыз:
1. Сол жақ бүйірлік панельдегі **Параметрлер** (Settings, тісті дөңгелек белгішесі) басыңыз.
2. **Желі параметрлері** (Network Settings) бөліміне өтіңіз.
3. **Жүктеу көзі / Айна** (Download Source / Mirror) тармағын табыңыз.
4. Оны **Әдепкі** (Default) күйінен **BMCLAPI** немесе **MCBBS** күйіне ауыстырыңыз (олар ресми файлдарды толық көшіретін сенімді жүктеу көздері).
:::

:::info Прокси параметрлері
Егер желіңізде белгілі бір серверлерге кіру шектелген болса, проксиді тікелей лаунчер ішінде конфигурациялай аласыз:
1. **Параметрлер** -> **Желі параметрлері** бөліміне өтіп, прокси параметрлерін табыңыз.
2. SOCKS5 немесе HTTP прокси-серверінің мекенжайын енгізіңіз.
3. Қосылымды тексеріңіз.
:::

## 📦 2. Мод немесе модпак жүктелмейді (CurseForge API шектеуі)

### Симптомдары
* CurseForge-тен модпак немесе жеке мод жүктеген кезде кейбір модтар жүктелмейді және олардың жанында ескерту белгісі пайда болады.
* "Үшінші тарап лаунчерлеріне жүктеу шектеуі" туралы хабарлама көрсетіледі.

### Себебі
CurseForge-тегі кейбір мод авторлары ойыншыларды өздерінің ресми веб-парақшаларына кіруге мәжбүрлеу үшін үшінші тарап лаунчерлеріне API арқылы жүктеу мүмкіндігін өшіріп тастайды.

### Шешімі
XMCL бұл мәселені шешу үшін жетіспейтін файлдарды қолмен жүктеуге мүмкіндік береді:
1. Лаунчердің тапсырмалар менеджеріндегі (жоғарғы оң жақта) сәтсіз аяқталған тапсырманың мәліметтерін ашыңыз.
2. Жүктеуі сәтсіз аяқталған модтың жанындағы сілтемені басып, оның жүктеу бетін шолғышта ашыңыз.
3. `.jar` файлын шолғыш арқылы қолмен жүктеңіз.
4. Жүктелген `.jar` файлын тікелей лаунчер терезесіне **сүйреп апарыңыз** (немесе оны осы профильдің `mods`資料夾на салыңыз).
5. XMCL оны автоматты түрде танып, орнатуды аяқтайды.

## 🔍 3. Мод CurseForge сайтында бар, бірақ лаунчер іздеуінде жоқ

### Симптомдары
* Лаунчерден мод іздегенде, ол ресми CurseForge сайтында анық болса да, «Нәтиже табылмады» деген хабарлама шығады.

### Себебі
CurseForge платформасы мод авторларына өз жұмыстарына **үшінші тарап қолданбаларының (API) қатынасуын өшіруге** мүмкіндік береді. Егер бұл өшірілген болса, CurseForge API (оны XMCL іздеу үшін пайдаланады) бұл модты іздеу нәтижелерінде көрсетпейді.

### Шешімі
1. Шолғышты ашып, [CurseForge](https://www.curseforge.com/minecraft/search) сайтындағы мод бетіне өтіңіз.
2. Компьютеріңізге `.jar` файлын жүктеу үшін **Download** түймесін басыңыз.
3. XMCL-ді ашып, қажетті ойын профилін (инстансты) таңдаңыз.
4. Жүктелген `.jar` файлын тікелей лаунчер терезесіне **сүйреп апарыңыз (Drag-and-drop)**. XMCL оны автоматты түрде таңдалған профильдің `mods` папкасына орнатады.

## 📦 4. Импортталған модпактар «жоғалып» кетеді немесе бос болып көрінеді

### Симптомдары
* Сіз модпактың `.zip` немесе `.mrpack` файлын лаунчерге сүйреп апардыңыз, бірақ оны ағымдағы ойын профилінен таба алмайсыз немесе модтар тізімі бос.

### Себебі
1. **Жаңа инстанс жасау**: XMCL модпактарды ағымдағы белсенді профиліңізге қоспайды. Оның орнына ол осы модпак үшін **мүлдем жаңа профиль (инстанс)** жасайды.
2. **Фондық жүктеу жұмыстары**: Модпак файлдары орынды үнемдеу үшін нақты `.jar` мод файлдарын қамтымайды (оларда тек тізім болады). Импорттағаннан кейін XMCL барлық қажетті модтарды жүктеу үшін фондық тапсырманы іске қосады. Бұл тапсырма аяқталғанша модтар тізімі бос болып көрінуі мүмкін.

### Шешімі
1. **Профильді ауыстырыңыз**: Барлық қолжетімді профильдерді көру үшін бүйірлік панельді немесе профиль ауыстырып-қосқышты басыңыз. Онда модпак атауымен жаңа профильді тауып, оны таңдаңыз.
2. **Тапсырмалар менеджерін тексеріңіз**: Модпак файлдарының жүктелуі аяқталғанын тексеру үшін лаунчердің жоғарғы оң жақ бұрышындағы тапсырмалар белгішесін басыңыз. Ойынды іске қоспас бұрын жүктеудің толық аяқталуын күтіңіз.

## 🔄 5. Файлдардың зақымдалуына байланысты шексіз жүктеу циклі (Бақылау сомасы сәйкес келмеуі)

### Симптомдары
* Лаунчер бір файлды (кітапхана немесе ресурс) оның зақымдалғанын айтып, қайта-қайта жүктей береді.
* Файлдарды тексеру қатемен аяқталғандықтан ойын іске қосылмайды.

### Себебі
Файлды жүктеу үзілген және зақымдалған файл сіздің локалдық кэшіңізде бұғатталып қалған, бұл лаунчерге оны жаңа файлмен дұрыс ауыстыруға кедергі келтіреді.

### Шешімі
1. Лаунчер диагностикасында немесе журналдарында көрсетілген зақымдалған файлдың жолын табыңыз (мысалы, `libraries/org/lwjgl/...`).
2. Ойын профилінің деректер папкасын ашыңыз (профиль жұмыс үстелінің жоғарғы оң жағындағы папка белгішесін басыңыз).
3. Қатеде көрсетілген жол бойынша өтіп, зақымдалған файл бар **папканы толығымен өшіріңіз**.
4. **«Түзету»** (Repair) түймесін басыңыз немесе ойынды қайта іске қосыңыз. Лаунчер файлдың таза әрі бүтін көшірмесін жүктейді.

## ☕ 6. Ойын іске қосылғаннан кейін бірден жабылып қалады (Java нұсқасынос сәйкес келмеуі)

### Симптомдары
* Ойын іске қосылады, бірақ бірден `1` немесе `-1` қате кодымен жабылады.
* Журналдарда `UnsupportedClassVersionError` қатесі немесе "Java табылмады" хабарламасы пайда болады.

### Себебі
Minecraft-тың әрбір нұсқасы белгілі бір Java (JDK) нұсқасын қажет етеді. Дұрыс емес нұсқаны пайдалану ойынның жабылып қалуына әкеледі.

### Шешімі
XMCL-де сізге қажетті JDK нұсқаларын автоматты түрде жүктей алатын ішкі Java менеджері бар.

:::warning Java нұсқаларының үйлесімділігі
Профильдің дұрыс Java нұсқасын пайдаланып жатқанына көз жеткізіңіз:
* **Minecraft 1.12.2 және одан ескі нұсқалар:** Java 8
* **Minecraft 1.16 - 1.17:** Java 16 / 17
* **Minecraft 1.18 - 1.20.4:** Java 17
* **Minecraft 1.20.5+:** Java 21
:::

#### XMCL-де Java-ны қалай таңдауға болады:
1. Ойын профилінің параметрлеріне өтіңіз («Ойнау» батырмасының жанындағы тісті дөңгелек белгішесі).
2. **Java** бөлімін табыңыз.
3. Таңдау өрісін басыңыз. XMCL жүйеңіздегі барлық анықталған Java нұсқаларын көрсетеді және үйлесімді нұсқаларды жасыл белгімен белгілейді.
4. Қажетті нұсқа болмаса, **«Java орнату»** түймесін басыңыз — лаунчер ұсынылған нұсқаны автоматты түрде жүктейді.

## 📑 7. Launcher мүлдем ашылмайды немесе қара экранды көрсетеді

### Симптомдары
* Лаунчер белгішесін екі рет басу ешқандай нәтиже бермейді.
* Лаунчер терезесі ашылады, бірақ толығымен қара болып қалады.

### Шешімі
Ақаулықтың себебін білу үшін журнал файлдарын таба аласыз:
1. Компьютеріңіздегі лаунчердің деректер папкасына өтіңіз:
   * **Windows:** `%appdata%\xmcl`
   * **macOS:** `~/Library/Application Support/xmcl`
   * **Linux:** `~/.config/xmcl`
2. `logs` папкасын ашып, ең соңғы `main.log` файлын табыңыз.

## 📋 8. Диагностикалық есепті жасау (Алдымен ұсынылатын қадам)

Журнал файлдарын қолмен іздемес бұрын, лаунчерде тікелей **диагностикалық есеп** жасауды ұсынамыз. Бұл лаунчер мен ойын журналдарын және жүйелік ақпаратты бір файлға автоматты түрде жинақтауға мүмкіндік береді, бұл әзірлеушілерге немесе қоғамдастыққа мәселеңізді тезірек шешуге көмектеседі.

### Есепті қалай жасауға болады:
1. Лаунчердің жоғарғы панеліндегі **Көмек** (Help) немесе **Кері байланыс** батырмасын басыңыз.
2. Қажетті журналдарды жинау үшін **Диагностикалық есеп жасау** (Generate Report) опциясын таңдаңыз.

## 📑 9. Лаунчер және ойын журналдарын (логтарын) қалай талдауға болады

Журналдарды қолмен іздегіңіз келсе, олар сізге нақты не болғанын түсінуге көмектеседі. Төменде осы файлдарды қалай табуға және жиі кездесетін мәселелерді шешуге болатыны сипатталған.

### 🔍 Журнал файлдарын қалай табуға болады

Қатенің түріне байланысты (лаунчердің бұзылуы немесе ойынның крашы), сізге әртүрлі журнал файлдары қажет:

#### А. Лаунчер журналдары (`main.log`)
Лаунчер бұзылғанда, файлдар жүктелмегенде, желі қателері немесе жүйеге кіру мәселелері туындағанда:
- **Windows:** `Win + R` пернелерін басып, `%appdata%\xmcl\logs` енгізіп, Enter басыңыз.
- **macOS:** `~/Library/Application Support/xmcl/logs` папкасына өтіңіз.
- **Linux:** `~/.config/xmcl/logs` папкасына өтіңіз.
- Ең соңғы `main.log` файлын табыңыз.

#### Б. Ойын журналдары (`latest.log` және краш есептері)
Модтар қақтығысы, Minecraft крашы, өнімділік немесе Java мәселелері үшін:
- Лаунчердегі ойын профилінің картасын ашыңыз.
- Оның директориясын ашу үшін профиль жұмыс үстелінің жоғарғы оң жағындағы **Папка** белгішесін басыңыз.
- `logs` папкасына өтіп, `latest.log` файлын ашыңыз.
- Егер ойын кенеттен жабылып қалса (краш болса), `crash-reports` папкасына өтіп, ең жаңа мәтіндік файлды табыңыз (атау форматы `crash-YYYY-MM-DD_HH.MM.SS-client.txt`).

---

### 🛠 Журналдарды талдау және жиі кездесетін мәселелерді шешу

Журнал файлын кез келген мәтіндік редакторда ашып, негізгі қателерді табу үшін іздеуді (`Ctrl + F`) пайдаланыңыз:

#### 🔴 1-жағдай: Жедел жадтың жетіспеушілігі (Out of Memory)
- **Не іздеу керек:** `java.lang.OutOfMemoryError: Java heap space` немесе шығу коды `Exit code: -805306369`.
- **Себебі:** Ойынды таңдалған модтар санымен іске қосу үшін жеткілікті жедел жад (RAM) бөлінбеген.
- **Қалай түзету керек:**
  1. Ойын профилінің параметрлеріне өтіңіз (түйменің жанындағы тісті дөңгелек белгішесі).
  2. Бетті төмен қарай **Java** бөліміне дейін айналдырыңыз.
  3. **Мин. жад** және **Макс. жад** параметрлерін арттырыңыз (мысалы, Максималды жадты `4096` немесе `6144` МБ етіп орнатыңыз).

#### 🔴 2-жағдай: Модтар қақтығысы немесе тәуелділіктердің жоқтығы
- **Не іздеу керек:** `Mixin transformation failed`, `DependencyResolutionException` немесе `Requires mod 'fabric' (version X or later), but only version Y is installed` сияқты хабарламалар.
- **Себебі:** Модтарыңыздың біріне папкада жоқ қосымша модты (тәуелділікті) орнату қажет немесе екі мод өзара үйлеспейді.
- **Қалай түзету керек:** Қате мәтінін мұқият оқып шығыңыз. Әдетте онда жетіспейтін модтың атауы көрсетіледі. Оны жүктеп алып, `mods` папкасына салыңыз немесе үйлесімсіз модты өшіріңіз.

#### 🔴 3-жағдай: Үйлесімсіз Java нұсқасы
- **Не іздеу керек:** `java.lang.UnsupportedClassVersionError: ... has been compiled by a more recent version of the Java Runtime`.
- **Себебі:** Сіз таңдалған Minecraft нұсқасына немесе зерттемесіне сәйкес келмейтін Java нұсқасын пайдаланып жатырсыз (мысалы, Minecraft 1.20 үшін Java 8 пайдалану).
- **Қалай түзету керек:** Профиль параметрлерінде **Java** бөліміне өтіп, **«Java орнату»** түймесін басыңыз — лаунчер ұсынылған нұсқаны өзі жүктейді.

#### 🔴 4-жағдай: Бейнекарта драйверінің бұзылуы (OpenGL)
- **Не іздеу керек:** `GLFW error 65542: WGL: The driver does not seem to support OpenGL` немесе `Pixel format not accelerated`.
- **Себебі:** Бейнекарта драйверлеріңіз ескірген немесе ойын дискретті бейнекартаның орнына процессордың кірістірілген графикасында іске қосылуға тырысуда.
- **Қалай түзету керек:** Бейнекартаңыздың драйверлерін (NVIDIA, AMD немесе Intel) өндірушінің ресми сайтынан соңғы нұсқасына дейін жаңартыңыз. Ноутбуктерде жүйелік параметрлерде лаунчер мен Java үшін жоғары графикалық өнімділік режимі таңдалғанына көз жеткізіңіз.
