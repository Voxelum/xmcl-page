# Démarrer avec la localisation

## Prérequis

- [Git](https://git-scm.com/). Vous **devez** installer Git pour commencer.
- [VSCode](https://code.visualstudio.com/). L'éditeur fortement recommandé pour les traducteurs. Il dispose d'un outil i18n intégré pour vous aider à trouver les clés de traduction.
- [Node.js](https://nodejs.org/). Nécessaire pour tester vos traductions localement (installez la dernière version).
- [pnpm](https://pnpm.io/installation). Nécessaire pour installer et lancer le projet localement.

## Démarrage

### Fork & Clone <Badge type="danger">Requis</Badge>

Vous devez forker et cloner le projet avec Git :

```bash
git clone --recurse-submodules https://github.com/votre-id/x-minecraft-launcher
```

### Installation <Badge type="danger">Requis</Badge>

Installez les dépendances avec pnpm :

```bash
pnpm install
```

> Si vous utilisez `corepack`, vous pouvez lancer `corepack enable .` pour activer pnpm.

Si vous rencontrez une erreur indiquant que des modules sont introuvables, assurez-vous d'avoir cloné avec l'option `--recurse-submodules`. Sinon, lancez :

```bash
git submodule update --init --recursive
```

puis relancez `pnpm install`.

### Trouver le fichier de langue

Les fichiers de langue au format YAML se trouvent sous :

- `xmcl-keystone-ui/locales`
- `xmcl-electron-app/main/locales`

Le nom du fichier correspond au code de langue (ex. `fr.yaml` pour le français).

### Utiliser VSCode avec l'extension i18n-ally <Badge type="tip">Recommandé</Badge> <Badge type="info">Optionnel</Badge>

Installez l'extension `i18n-ally` (`lokalise.i18n-ally`).

Dans l'onglet `PROGRESS` de cette extension, vous pouvez voir la progression de votre langue et le nombre de clés manquantes. Vous pouvez modifier les traductions directement dans l'éditeur.

### Ajouter une nouvelle langue <Badge type="info">Optionnel</Badge>

Si vous ajoutez une nouvelle langue, ajoutez son code et son libellé dans `assets/locales.json`.

Exemple pour le français (`fr`) :

```json
{
  "zh-CN": "简体中文",
  "zh-TW": "繁體中文",
  "en": "English",
  "ru": "Русский язык",
  "es-ES": "Español",
  "fr": "French"
}
```

Créez ensuite le fichier **yaml** correspondant avec le code de langue (ex. `fr.yaml`).
